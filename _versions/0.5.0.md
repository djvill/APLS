This is a "preview" version of APLS.
We have uploaded **all remaining transcript series**; however, we are still tweaking transcript data and refining the UI, so existing data is subject to change in minor ways.


#### New layers/attributes {#new-layers-attributes-{{ page.version }}}
{:.no_toc}

- Added new <span class="layer">prec_pause</span> and <span class="layer">prec_segment</span> layers. These are the counterpart to the existing <span class="layer">foll_pause</span> and <span class="layer">foll_segment</span> layers, and they should make it easier to specify search environments and export data.
  - Thanks to Meredith Tamminga for the idea for this layer!
- Added new <span class="layer">frequency_from_subtlex</span> layer, which tags words with their word frequency from the [SUBTLEX-US] corpus of subtitles from 8,388 films. The authors of SUBTLEX-US [argue][brysbaert and new 2009] that its frequency measurements represent an improvement over those from Kučera & Francis or [CELEX]. This layer specifically uses SUBTLEX-US's `Lg10CD` measure, the base-10 logarithm of the number of films that the word appears in, on the advice of Meredith Tamminga.
  - Thanks to Meredith Tamminga for the idea for this layer!
  
  {: .note}
  > SUBTLEX-US has separate entries for clitics like _'s_, meaning there are no frequency counts for cliticized words like _what's_.
  > As a result, words with clitics have no <span class="layer">frequency_from_subtlex</span> annotation, including some high-frequency words like _it's_ (6,920 tokens in APLS).
  > However, CELEX lacks entries for high-frequency fillers like _hmm_ (11,992 tokens in APLS).
  > Here's how the numbers stack up for the 518,785 word tokens for which <span class="layer">hesitation</span> is `False` (since hesitations like _wh~_ wouldn't be expected to be in either reference corpus):

- Added new <span class="layer">word_index</span> layer, which labels each word's index (e.g., 1st, 5th, 270th) for that participant in that transcript:
  {% include screengrab.html src="transcript/word_index.png" %}
  This can be useful for assessing effects of recency (e.g., how many words ago did this speaker use the same word?)
  - Thanks to James Stratton for the idea for this layer!
- Added new <span class="participant-attr">word_count</span> participant attribute. This is useful for things like calculating and comparing speakers' density of using particular words (e.g., _um_ and _uh_).
  - Thanks to Scott Kiesling for the idea for this attribute!
- Added new <span class="participant-attr">tasks</span> participant attribute, which lists the [interview task(s)]({{ '/doc/pittsburgh-interviews' | relative_url }}) that each main participant has in APLS. This is equivalent to listing the <span class="transcript-attr">type</span>s of all the transcripts in which that participant appears. This attribute is useful for contextualizing why some participants have low <span class="participant-attr">word_count</span>s. It's also useful for filtering participants (if, for example, you want to search only participants who have `interview`, `reading`, **and** `pairs` tasks).

#### User interface changes {#user-interface-changes-{{ page.version }}}
{:.no_toc}

Below, you can click the GitHub icon <img src="{{ '/assets/img/GitHub_Invertocat_Black.svg' | relative_url }}" style="height: 1rem;"/> to view code changes on GitHub.

- Pitch measurement on the <span class="apls-page">Process with Praat</span> page {% include github-link.html path="nzilbb/labbcat-server/pull/158" %}
  - As of 2023, the Praat authors [recommend](https://praat.org/manual/how_to_choose_a_pitch_analysis_method.html) Praat's filtered autocorrelation method for pitch analysis instead of raw autocorrelation (which had been the recommended method since 1993). As a result, this is now the default pitch analysis method in APLS. However, users can still use raw autocorrelation (now a drop-down menu option) or specify a custom Praat command if they wish.
    {% include compare-ui.html file="0.5.0/praat-pitch" %}

- In the _CSV Export_ panel of the <span class="apls-page">Matches</span> page, the "linked annotations" counter now defaults to returning `all` linked annotations, giving users a fuller picture of linked annotations. {% include github-link.html path="djvill/labbcat-server/commit/83f26dc" %}

- Symbol picker on the <span class="apls-page">Search</span> page
  - Previously, syllabic consonants were a subcategory under the _CONSONANTS_ category. This didn't make sense for layers like <span class="layer">foll_segment</span>, since the onset of a syllabic consonant is more vowel-like. Now, syllabic consonants are under _VOWELS_ for <span class="layer">foll_segment</span> and _CONSONANTS_ for <span class="layer">prec_segment</span>; for all other layers, syllabic consonants are their own category, since we can't assume _a priori_ whether it makes sense to categorize them as vowels, consonants, or neither.{% comment %}No github-link here because it's actually a change to the data (MySQL table `label_option`).{% endcomment %}
    - To search for a single segment that is either a non-syllabic consonant or a syllabic consonant: type `[`, click both _CONSONANTS_ and _SYLLABIC CONSONANTS_, then type `]` to close the [character class]({{ '/doc/regex#square-brackets-' | relative_url }}). This will result in the pattern `[[pbtdkgfvTDszSZhJ_mnNlrwj][FHP]]`.
  - Previously, for technical reasons, there were no symbols for /ɔ(ɹ)/ and /ɚ/. Now, these are present. {% include github-link.html path="djvill/labbcat-server/commit/7a9e28a" %}

- On individual <span class="apls-page">Transcript</span> pages, the [word menu]({{ '/doc/view-transcript#word-menu' | relative_url }}) has a new _Copy transcript text_, which copies the <span class="layer">word</span> annotations for that <span class="layer">utterance</span> to the user's clipboard. {% include github-link.html path="nzilbb/labbcat-server/pull/161" %}

- On the <span class="apls-page">Search</span> page, [input boxes]({{ '/doc/search#search-matrix-and-pattern-inputs' | relative_url }}) that accept [regular expressions]({{ '/doc/regex' | relative_url }}) do a better job of checking for errors in regular expressions. In particular, these input boxes now accept regular expressions with nested square brackets (like `[[7][8]]`) as valid and reject regular expressions with unmatched `[` or empty `[]`. {% include github-link.html path="nzilbb/labbcat-server/pull/163" %}

- Other user-experience "quality-of-life" improvements on the <span class="apls-page">Process with Praat</span> page {% include github-link.html path="nzilbb/labbcat-server/pull/158" %}
  - For example, previously, if you clicked [_Pitch_]({{ '/doc/extract-acoustics#extracting-pitch-measurements' | relative_url}}) or [_Intensity_]({{ '/doc/extract-acoustics#extracting-intensity-measurements' | relative_url}}) and configured advanced pitch/intensity settings, you might not have remembered that you also needed to select which measure(s) you wanted to extract. (This generally isn't an issue for formant measurement since _F1_ and _F2_ are selected by default.) Now, if you change any advanced pitch/intensity settings but don't select a measure, APLS will ask if you're sure you want to proceed. {% include github-link.html path="djvill/labbcat-server/commit/7b846ce" %}
    {% include screengrab.html src="praat/forgot-measures.png" %}


#### Updates to existing layers/attributes {#updates-to-existing-layers-attributes-{{ page.version }}}
{:.no_toc}

- Removed about 6,000 annotations that didn't belong to any transcript: 5,898 <span class="layer">word</span> annotations, 8 <span class="layer">orthography</span> annotations, and 10 <span class="layer">dictionary_phonemes</span> annotations. These were probably holdovers from earlier versions of the corpus when things were more experimental. This won't affect any data users might have downloaded; it mostly just affects the "total word count" stats for the corpus.
- Corrected errors in <span class="transcript-attr">right_channel_participants</span> for 96 transcripts (including all Hill District transcripts).
